# 生产环境部署指南

> TeamTalk 生产环境部署与运维文档。

## 架构概览

TeamTalk 采用单体架构，所有组件运行在一台服务器上：

```
┌──────────────────────────────────────────────┐
│               生产服务器                       │
│                                              │
│  TeamTalk Server (Ktor + Netty)              │
│  ├── HTTPS API ── 端口 443（需 SSL 证书）    │
│  ├── HTTP API  ── 端口 8080（无 SSL 时默认） │
│  └── TCP 长连接 ── 端口 5100                 │
│                                              │
│  PostgreSQL ── 端口 5432 (仅内网)            │
│  MinIO ── 端口 9000 / 9001 (仅内网)          │
└──────────────────────────────────────────────┘
```

推荐配置：2 核 4G 即可支撑万级用户规模。

## 一键部署

项目提供了 `deploy.sh` 脚本，在本地执行即可自动完成远程服务器的部署和升级：

```bash
# HTTP 部署（内部测试）
./deploy.sh root@192.168.1.100

# HTTPS 部署（生产环境，提供 SSL 证书）
./deploy.sh root@im.example.com --ssl-cert server.pem --ssl-key server.key

# 指定部署路径和 HTTPS 端口
./deploy.sh root@im.example.com --path /opt/teamtalk --ssl-cert cert.pem --ssl-key cert.key --ssl-port 443

# 升级（自动检测已有部署，保留数据和配置）
./deploy.sh root@im.example.com

# 升级时更新 SSL 证书
./deploy.sh root@im.example.com --ssl-cert new-cert.pem --ssl-key new-cert.key
```

脚本功能：
- 自动构建 → 上传 → Docker 基础设施部署 → 配置生成 → 服务注册 → 启动
- 首次部署生成随机密码（数据库、JWT、MinIO），写入 `env.sh`（权限 600）
- 支持 SSL：自动将 PEM 证书转换为 PKCS12，配置 Ktor HTTPS
- 升级时自动备份旧版本，保留 `data/`、`logs/`、`env.sh`、`conf/ssl/`
- 部署完成后自动健康检查（自动适配 HTTP/HTTPS）
- 支持 SSH 免密登录或 `sshpass` 密码认证

前置要求：
- 本地：JDK 17+、rsync、openssl（启用 SSL 时）
- 服务器：Docker + Docker Compose、JDK 17+

### SSL 证书

deploy.sh 接受 PEM 格式的证书和私钥（Nginx/Apache 格式），自动完成：

1. 本地将 PEM 转换为 PKCS12（密码随机生成）
2. 上传到服务器 `$DEPLOY_PATH/conf/ssl/teamtalk.p12`（权限 600）
3. 将 SSL 配置追加到 `env.sh`（不入 Git）

证书文件**不会**进入 Git 仓库，由部署者本地管理。升级时重新提供 `--ssl-cert/--ssl-key` 即可更新证书。

## 构建分发产物

```bash
./gradlew :server:buildServerDist
```

产物目录结构：

```
server/build/install/teamtalk-server/
├── bin/
│   ├── server                # Gradle 生成的标准启动脚本
│   ├── teamtalk.sh           # 后台启动脚本（PID 文件管理）
│   └── teamtalk-stop.sh      # 停止脚本
├── lib/                      # 依赖 JAR（非 fat JAR）
├── conf/
│   ├── application.conf      # 服务端配置（开发默认值）
│   └── logback.xml           # 日志配置
├── static/
│   ├── index.html            # 产品首页
│   └── downloads/            # 客户端安装包（手动上传）
├── data/                     # RocksDB 消息存储（运行时生成）
└── logs/                     # 日志文件（运行时生成）
```

## 配置参考

### 配置原则

`application.conf` 只包含开发默认值（HTTP 8080、无 SSL、默认密码）。所有生产配置通过 `env.sh` 环境变量覆盖，不修改配置文件。

优先级：**环境变量 (env.sh) > 配置文件默认值**。所有配置项都支持环境变量覆盖（HOCON `${?ENV_VAR}` 语法）。

### 配置项

| 配置项 | 默认值 | 环境变量 | 说明 |
|--------|--------|----------|------|
| HTTP 端口 | `8080` | `KTOR_PORT` | REST API 端口（设为空可关闭） |
| HTTPS 端口 | _(无)_ | `KTOR_SSL_PORT` | 启用后自动关闭 HTTP |
| SSL 证书库 | _(无)_ | `SSL_KEYSTORE` | PKCS12 文件路径 |
| SSL 证书库密码 | _(无)_ | `SSL_KEYSTORE_PASSWORD` | — |
| SSL 私钥密码 | _(无)_ | `SSL_PRIVATE_KEY_PASSWORD` | — |
| TCP 端口 | `5100` | `TCP_PORT` | 长连接推送端口 |
| 数据库 URL | `jdbc:postgresql://127.0.0.1:5432/teamtalk` | `DATABASE_JDBC_URL` | — |
| 数据库用户 | `postgres` | `DATABASE_USER` | — |
| 数据库密码 | `postgres` | `DATABASE_PASSWORD` | 生产环境务必修改 |
| JWT 密钥 | 内置默认值 | `JWT_SECRET` | 至少 256 位 |
| MinIO 端点 | `http://127.0.0.1:9000` | `MINIO_ENDPOINT` | — |
| MinIO AccessKey | `minioadmin` | `MINIO_ACCESS_KEY` | — |
| MinIO SecretKey | `minioadmin` | `MINIO_SECRET_KEY` | 生产环境务必修改 |
| MinIO Bucket | `teamtalk` | `MINIO_BUCKET` | — |
| 文件大小上限 | `52428800`（50MB） | `FILE_MAX_SIZE_BYTES` | — |

### env.sh 示例

deploy.sh 自动生成的 `/opt/teamtalk/env.sh`（权限 600）：

```bash
# TeamTalk 环境配置 — 自动生成
# 权限 600，请勿提交到版本控制
# 修改后执行: systemctl restart teamtalk

DATABASE_JDBC_URL="jdbc:postgresql://127.0.0.1:5432/teamtalk"
DATABASE_USER="teamtalk"
DATABASE_PASSWORD="<auto-generated>"
JWT_SECRET="<auto-generated-256bit>"
MINIO_ENDPOINT="http://127.0.0.1:9000"
MINIO_ACCESS_KEY="<auto-generated>"
MINIO_SECRET_KEY="<auto-generated>"
MINIO_BUCKET="teamtalk"
TK_LOG_DIR="/opt/teamtalk/data/logs"

# SSL 配置（--ssl-cert 部署时自动追加）
KTOR_SSL_PORT=443
SSL_KEYSTORE=/opt/teamtalk/conf/ssl/teamtalk.p12
SSL_KEYSTORE_PASSWORD=<auto-generated>
SSL_PRIVATE_KEY_PASSWORD=<auto-generated>
```

### 日志配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `TK_LOG_DIR` | `data/logs` | 日志目录（由 Environment 自动设置） |
| 日志文件 | `teamtalk.log` | 主日志文件 |
| 滚动策略 | 50MB/文件，保留 30 天，总量上限 1GB | 按大小+时间滚动 |
| 日志级别 | `INFO` | 可在 `conf/logback.xml` 中调整 |

## 数据目录

| 路径 | 说明 | 备份建议 |
|------|------|----------|
| `data/rocksdb/` | RocksDB 消息存储 | 定期备份 |
| `data/lucene-index/` | Lucene 全文索引 | 可重建 |
| `data/pgdata/` | PostgreSQL 数据 | 定期备份 |
| `data/minio/` | MinIO 对象存储 | 定期备份 |
| `data/logs/` | 应用日志 | 可归档后清理 |
| `conf/` | 配置文件 | 首次部署后备份 |
| `conf/ssl/` | SSL 证书（PKCS12） | 证书更新时备份 |
| `static/` | 首页 + 客户端安装包 | 可重建 |
| `env.sh` | 环境变量（含密码） | **务必备份** |

## 系统服务（systemd）

deploy.sh 自动注册 systemd 服务：

```bash
# 查看状态
systemctl status teamtalk

# 重启（修改 env.sh 后）
systemctl restart teamtalk

# 查看日志
journalctl -u teamtalk -f
```

## 客户端发布

使用 `build-release.sh` 统一构建客户端安装包：

```bash
# 构建所有平台（需在对应平台构建 Desktop 包）
./build-release.sh \
  --server-url https://im.example.com \
  --tcp-host im.example.com \
  --version 1.0.0

# 构建 Desktop + Android 并上传到服务器
./build-release.sh \
  --server-url https://im.example.com \
  --tcp-host im.example.com \
  --upload im.example.com

# 仅构建 Android
./build-release.sh --android-only --server-url https://im.example.com --tcp-host im.example.com
```

构建产物上传到服务器后，用户可通过首页 `https://im.example.com/` 下载。

### Android 签名

Release APK 需要签名密钥。生成后在 `local.properties` 中配置：

```bash
keytool -genkey -v -keystore teamtalk-release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias teamtalk
```

```properties
# local.properties
release.storeFile=teamtalk-release.jks
release.storePassword=<password>
release.keyAlias=teamtalk
release.keyPassword=<password>
```

## 备份与恢复

### PostgreSQL

```bash
# 备份
docker exec teamtalk-postgres-1 pg_dump -U teamtalk teamtalk > backup_$(date +%Y%m%d).sql

# 恢复
cat backup_20260403.sql | docker exec -i teamtalk-postgres-1 psql -U teamtalk teamtalk
```

### RocksDB 消息数据

```bash
# 停止服务后直接打包
/opt/teamtalk/bin/teamtalk-stop.sh
tar czf rocksdb_$(date +%Y%m%d).tar.gz /opt/teamtalk/data/rocksdb/
```

## 升级

```bash
# 一键升级（保留数据和配置）
./deploy.sh root@im.example.com

# 升级并更新 SSL 证书
./deploy.sh root@im.example.com --ssl-cert new-cert.pem --ssl-key new-cert.key

# 升级并上传新客户端安装包
./deploy.sh root@im.example.com --upload-packages
```
